\section{Harbor registry storage trên GCS}

Phần Harbor GCS storage được thể hiện qua các file sau:

\begin{itemize}
  \item \path{infra/opentofu/modules/harbor-registry-storage}: tạo bucket \texttt{sagelms-devsecops-harbor-registry}, GSA và bucket IAM.
  \item \path{infra/opentofu/envs/devsecops/outputs.tf}: xuất thông tin bucket, service account và Workload Identity handoff.
  \item \path{infra/k8s/devsecops/harbor/values-gcs.yaml}: cấu hình Harbor registry dùng backend GCS.
  \item \path{infra/k8s/devsecops/harbor/registry-pvc-to-gcs-job.yaml}: job copy dữ liệu registry từ PVC \path{/storage} sang GCS bằng \texttt{gcloud storage rsync}.
\end{itemize}

Điểm cần ghi chú khi nghiệm thu là Harbor metadata và registry blobs là hai lớp khác nhau. Database/metadata của Harbor có thể vẫn nằm trong runtime Harbor, còn image blobs được chuyển sang GCS backend để tăng độ bền dữ liệu artifact.

\placeholderfigure{Placeholder ảnh Harbor GCS backend}{Vị trí chèn ảnh Harbor Helm values hoặc bucket \path{gs://sagelms-devsecops-harbor-registry}; nếu đã migrate từ PVC, chèn thêm log job rsync hoặc đối chiếu object count trước/sau.}
